1. Etl Pipeline

  • ETL：Extract，Transform，Load；

  • RAG：Retrieval Augmented Generation；

  • ETL框架是RAG(检索增强生成)用例中数据处理的骨干；ETL流水线协调
    从原始数据源到结构化向量存储的流，确保数据处于AI模型检索的最佳格式；

  • RAG用例通过从data body中检索相关信息来增强生成模型能力的文本，
    以增强生成输出的质量和相关性；

3. DocumentReader

  • provide source of document from diverse(多种) origin；

public interface DocumentReader extends Supplier<List<Document>> {}

3.1. JsonReader

  • JsonReader以json格式解析document；

@Component
public class MyAiApp {

	//json document to load
	@Value("classpath:bike.json")
	private Resource resource;

	List<Document> loadJsonAsDocuments() {
		JsonReader jsonReader = new JsonReader(resource, "description");
		return jsonReader.get();
	}
}

3.2. TextReader

  • TextReader process plain text document

@Component
public class MyTextReader {

	//text document to load
    @Value("classpath:text-source.txt")
	private Resource resource;

	List<Document> loadText() {
		TextReader textReader = new TextReader(resource);
		textReader.getCustomMetadata().put("filename", "text-source.txt");

		return textReader.get();
    }
}

3.3. PagePdfDocumentReader

  • PagePdfDocumentReader使用PdfBox库解析pdf文档

@Component
public class MyPagePdfDocumentReader {

	List<Document> getDocsFromPdf() {

		var pdfReader = new PagePdfDocumentReader("classpath:/sample.pdf",
				PdfDocumentReaderConfig.builder()
					.withPageTopMargin(0)
					.withPageExtractedTextFormatter(ExtractedTextFormatter.builder()
						.withNumberOfTopTextLinesToDelete(0)
						.build())
					.withPagesPerDocument(1)
					.build());

		return pdfReader.get();
    }

}

3.4. ParagraphPdfDocumentReader

  • ParagraphPdfDocumentReader使用pdf catalog(e.g. TOC)信息将输入的pdf
    拆分成text paragraph，并paragraph输出一个文档，另并非所有pdf文档都包含pdf目录；

@Component
public class MyPagePdfDocumentReader {

	List<Document> getDocsFromPdfwithCatalog() {

        new ParagraphPdfDocumentReader("classpath:/sample.pdf",
                PdfDocumentReaderConfig.builder()
                    .withPageTopMargin(0)
                    .withPageExtractedTextFormatter(ExtractedTextFormatter.builder()
                        .withNumberOfTopTextLinesToDelete(0)
                        .build())
                    .withPagesPerDocument(1)
                    .build());

		return pdfReader.get();
    }
}

4. DocumentTransformer

  • Transform batch of document as part of processing workflow；

public interface DocumentTransformer
	extends Function<List<Document>, List<Document>> {}

4.1. TextSplitter

  • The TextSplitter是抽象基类，以帮助divide document
    to fit AI model context window；

4.2. TokenTextSplitter

  • 拆分文档，同时保持token级别完整性；

4.3. ContentFormatTransformer

  • 确保所有文档内容格式一致uniform content format；

4.4. KeywordMetadataEnricher

  • 使用基本的关键字元数据增强Augment文档；

4.5. SummaryMetadataEnricher

  • 用摘要元数据(summarization metadata)丰富文档
    (Enriche文档)，以增强检索功能(enhanced retrieval)
