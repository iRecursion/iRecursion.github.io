1. Intro

  • 人类同时通过多种数据输入模式(mode)处理知识，学习方式learn way和体验experience
    都是多模态multimodal，不只是视觉vision、听觉audio和文本text；

  • process knowledge，simultaneously across multiple mode of data input；

  • 所有自然联系的事物都应该结合起来教授，all things that are
    naturally connected ought to be taught in combination；

  • 与这些原则相反，过去对ML的方法往往侧重于为处理单一模态而量身定制的专门模型，
    如为文本转语音或语音转文本等任务开发的音频模型，为对象检测和分类等任务开发的CV模型；

  • 如GPT-4 Vision，Vertex AI Gemini Pro Vision，Claude3，
    LLaVA和balklava能接受文本，图像，音频，视频的多种输入，并生成文本响应；

2. Multimodality

  • 多模态：模型能同时simultaneously理解和处理来自各种来源various source
    的信息，包括文本text、图像image、音频audio和其它数据格式；

  • 消息的内容字段主要用作文本输入，而可选的媒体字段(media)允许添加
    一或多个不同形式的additional content，如图像、音频和视频；

  • MimeType指定模态类型，depending on使用的LLM，
    媒体的数据字段可以是编码的原始媒体内容或内容的URI；

  • 媒体字段(media field)目前仅适用于用户输入消息(如UserMessage)，
    它对系统消息没意义，doesn’t hold significance for system msg；

  • 包含LLM响应的AssistantMessage仅提供文本内容，要生成非文本媒体输出，
    应用专用的单模态models.*之一；

  • 将图片作为输入，并要求LLM解释它在图片中看到的内容：

byte[] imageData = new ClassPathResource("/multimodal.png").getContentAsByteArray();

var userMessage = new UserMessage(
	// content
	"Explain what do you see in this picture?",
	// media
	List.of(new Media(MimeTypeUtils.IMAGE_PNG, imageData)));

ChatResponse response = chatModel.call(new Prompt(List.of(userMessage)));
