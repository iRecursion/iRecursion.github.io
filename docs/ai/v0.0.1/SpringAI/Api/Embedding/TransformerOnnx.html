1. Intro

  • TransformersEmbeddingClient是EmbeddingClient的实现，
    它使用选定的句子转换器在本地计算句子嵌入，locally compute sentence
    embedding using selected sentence transformer；

  • 它使用预训练的transformer模型，序列化为Onnx格式，用Deep Java库和
    微软Onnx Java运行时库来运行Onnx模型并计算Java中的embeddings；

  • EmbeddingClient，TransformersEmbeddingClient；

  • https://djl.ai
    https://onnx.ai
    https://onnxruntime.ai/docs/get-started/with-java.html

  • https://www.sbert.net/examples/applications/computing-embeddings/README.html#sentence-embeddings-with-transformers
    https://www.sbert.net

2. Serialize

  • 将Tokenizer和Transformer模型序列化为Onnx格式；
    方可在Java中运行，使用optimum-cli序列化；

python3 -m venv venv

source ./venv/bin/activate

pip install --upgrade pip

pip install optimum onnx onnxruntime

optimum-cli export onnx --generative \
	sentence-transformers/all-MiniLM-L6-v2 onnx-output-folder

  • 导出(export)sentence-transformers/all-MiniLM-L6-v2
    transformer 到into onnx-output-folder文件夹；

  • 稍后使用embedding client来include tokenizer.json和model.onnx文件；
    代替(in place of)all-MiniLM-L6-v2，我们可选择任何
    hugging face transformer identifier或提供直接文件路径；

3. Usage

  • Using Onnx Transformer model

<dependency>
  	<groupId>org.springframework.ai</groupId>
  	<artifactId>spring-ai-transformers</artifactId>
</dependency>

  • 创建新的TransformersEmbeddingClient实例；

  • 并使用setTokenizerResource(tokenizerJsonUri)
    和setModelResource(modelOnnxUri)
    设置导出的tokenizer.json和model.onnx文件的URI；

  • 支持的URI schema类型：A：classpath:，B：file:，C：https:

  • 若未明确设置类型，TransformersEmbeddingClient默认为
    sentence-transformers/all-MiniLM-L6-v2

var embeddingClient = new TransformersEmbeddingClient();

var jsonPath = "classpath:/onnx/all-MiniLM-L6-v2/tokenizer.json"
var onnxPath = "classpath:/onnx/all-MiniLM-L6-v2/model.onnx"


//可选，默认：jsonPath
embeddingClient.setTokenizerResource(jsonPath);

//可选，默认：onnxPath
embeddingClient.setModelResource(onnxPath);

//可选，默认：${java.io.tmpdir}/spring-ai-onnx-model
//only http/https resource are cached by default
embeddingClient.setResourceCacheDirectory("/tmp/onnx-zoo");

//(optional)set tokenizer padding if you see an errors like:
//"ai.onnxruntime.OrtException: Supplied array is ragged, ..."
embeddingClient.setTokenizerOptions(Map.of("padding", "true"));

embeddingClient.afterPropertiesSet();

List<List<Double>> embeddings = embeddingClient
	.embed(List.of("Hello world", "World is big"));

  • 手动创建时，必须在设置属性后和使用客户端前调用afterPropertiesSet()

  • 第一个embed()调用下载大型onnx模型并将其缓存在本地文件系统上，首次调用需较长时间，
    用setResourceCacheDirectory(<path>)设置onnx模型存储的本地文件夹，
    默认缓存文件夹：${java.io.tmpdir}/spring-ai-onnx-model

  • 将TransformersEmbeddingClient创建为Bean更方便convenient(也是首选)，
    然后就不必手动调用afterPropertiesSet()；

@Bean
public EmbeddingClient embeddingClient() {
	return new TransformersEmbeddingClient();
}

4. Starter

  • Transformer Embedding Spring Boot Starter

  • 使用spring.ai.embedding.transformer.*配置

<dependency>
   	<groupId>org.springframework.ai</groupId>
   	<artifactId>spring-ai-transformers-spring-boot-starter</artifactId>
</dependency>
spring:
  ai:
    embedding:
      transformer:
        tokenizer:
          padding: true
          uri: https://huggingface.co/intfloat/e5-small-v2/raw/main/tokenizer.json
        onnx:
          modelUri: https://huggingface.co/intfloat/e5-small-v2/resolve/main/model.onnx

  • 若抛出异常：Caused by: ai.onnxruntime.OrtException:
    Supplied array is ragged,..,需启用tokenizer padding

  • spring.ai.embedding.transformer.tokenizer.options.padding=true

5. Property

Property Description Default Value

spring.ai.embedding.transformer.enabled

启用Transformer Embedding client

true

spring.ai.embedding.transformer.tokenizer.uri

Onnx引擎创建的预训练的HuggingFaceTokenizer的URI，如tokenizer.json

onnx/all-MiniLM-L6-v2/tokenizer.json

spring.ai.embedding.transformer.tokenizer.options

HuggingFaceTokenizer选项：addSpecialTokens,modelMaxLength,
truncation,padding,maxLength,stride,padToMultipleOf

空

spring.ai.embedding.transformer.cache.enabled

启用remote Resource caching

true

spring.ai.embedding.transformer.cache.directory

cache remote resource的路径，如onnx模型

${java.io.tmpdir}/spring-ai-onnx-model

spring.ai.embedding.transformer.onnx.modelUri

现有的Existing，经预训练的onnx模型

onnx/all-MiniLM-L6-v2/model.onnx

spring.ai.embedding.transformer.onnx.gpuDeviceId

GPU deviceId to execute on，仅在大于零时适用，否则忽略

-1

spring.ai.embedding.transformer.metadataMode

指定(文档内容和元数据)的哪些部分将用于计算嵌入computing embedding

NONE
