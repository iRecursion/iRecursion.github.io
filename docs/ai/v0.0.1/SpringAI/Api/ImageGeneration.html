1. Introduction

  • Image Generation Api用于与专门用于图像生成的各种AI模型交互，
    使工程师在不同的图像相关模型间切换，而尽量少改代码；

  • 另输入封装ImagePrompt和输出处理的ImageResponse等类支持，
    图像生成Api将与专用于图像生成的AI模型的通信统一，管理请求准备和响应解析的复杂性；

  • Image Generation Api构建在Generic Model Api之上，提供特定于图像的抽象和实现；

2. Image Client

@FunctionalInterface
public interface ImageClient extends ModelClient<ImagePrompt,ImageResponse> {

	ImageResponse call(ImagePrompt request);

}

2.1. ImagePrompt

  • ImagePrompt是ModelRequest，封装ImageMessage对象
    和可选模型请求选项的列表，optional model request option

public class ImagePrompt implements ModelRequest<List<ImageMessage>> {

    private final List<ImageMessage> messages;

	private ImageOptions imageModelOptions;

    @Override
	public List<ImageMessage> getInstructions() {...}

	@Override
	public ImageOptions getOptions() {...}

}

2.2. ImageMessage

  • ImageMessage类封装要使用的文本及文本在影响生成的图像时应该具有的权重，
    对支持权重的模型，可正positive可负negative

public class ImageMessage {

	private String text;

	private Float weight;

    public String getText() {...}

	public Float getWeight() {...}

}

2.3. ImageOptions

  • 可传递给图像生成模型的选项，ImageOptions类扩展ModelOptions接口，用于定义可传递给AI模型
    的几个可移植选项；另每个特定于模型的ImageClient实现都可有自己的选项，这些选项可传递给AI模型；

  • 如OpenAI图像生成模型有自己的选项，如质量quality、风格style等；允许工程师
    在启动应用时使用特定于模型的选项，然后在运行时使用ImagePrompt覆盖这些选项；

public interface ImageOptions extends ModelOptions {

	Integer getN();

	String getModel();

	Integer getWidth();

	Integer getHeight();

	//openai：url or base64，stability ai：byte[] or base64
	String getResponseFormat(); //

}

	private final List<ImageGeneration> imageGenerations;

	@Override
	public ImageGeneration getResult() {
		//get the first result
	}

	@Override
	public List<ImageGeneration> getResults() {...}

	@Override
	public ImageResponseMetadata getMetadata() {...}

}

  • ImageResponse类hold AI模型的输出，每个ImageGeneration
    实例都包含单个提示可能potentially产生的多个输出中的一个；

  • ImageResponse类还携带有关AI模型响应的ImageResponseMetadata元数据；
