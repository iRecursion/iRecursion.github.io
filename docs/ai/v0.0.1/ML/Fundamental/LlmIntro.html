  • 大语言模型：LargeLanguage Model

  • 大模型生成结果：根据上下文，猜下一个词的概率，此算法叫transformer；
    OpenAI的接口叫Completion，如Spring AI中叫Chat Completion Api；

import openai
import os

from dotenv import load dotenv,
	find dotenv = load dotenv(find dotev())

openai.api key = os.getenv('OPENAI API KEY')
prompt ="今天朕要"

response = openai.Completion.create(
	model="gpt-3.5-turbo-instruct"
	prompt = prompt,
	max tokens = 100
	temperature = 0.5,
	stream = True
)

for chunk in response:
	print(chunk.choices[0].text,end='')

  • temperature：控制输出的随机性randomness，温度越大，思维越广；

  • 范围0~2，默认1或0.7，取决于选择的模型，越高时更多随机输出，
    越接近零，则值的确定性更强deterministic

2. Token

  • token，是大模型处理的最小单位，可能是一个字或单词，如我，I，也可能是中文词，
    如我们，major in，还可能是半个或三分之一个汉字或单词，想想Unicode的存储方式；

  • 大模型在开训前，需先训练tokenizer模型，把所有的文本切成token；

3. Business Architecture

  • 大模型技术：A：训练基础大模型和B：建造大模型应用；

  • 大模型应用技术：门槛低，落地难；

3.1. Embedding

  • Embedding：此模式下，人占绝大多数，AI只做极少数部分的任务；

  • 系统有很多任务，只有某一环节/任务嵌入(参与)进来，如人脸识别

3.2. Copilot

  • Copilot：副驾驶，人和AI各占一半，两边协作同时把任务完成，
    最后由人拍板，啥可行，啥不可行，AI帮助我们提升效率；

3.3. Agent

  • 智能体模式，完全由AI来决定接下来要干啥，人类似于管理者，只提出问题，
    和最终拍板负责，其中的所有细节都由团队成员/AI负责完成；

4.3. Embedding + Vector DB

  • Embedding：主要解决相似度判断的问题，如谐音梗联想，
    具体可参考Spring AI Vector Store章节；

Entry Memo

Embedding

把文字转换为更易于相似度计算的编码，此编码称为向量

向量数据库

把向量存储，方便查找

向量搜索

根据输入向量，找到相似的向量

场景案例

再结合题目组成答案，打小抄，而结果也可能不好；
