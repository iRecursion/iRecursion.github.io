1. Intro

  • ML：将图像，视频，音频，文本等转换为数据，编写模型，然找出数字的模式Pattern；

  • https://paperswithcode.com/trends

2. When

  • problem with long rule list，当传统(approach) fail，ML/DL适应；

  • continually change environment，DL可adapt/learn to new scenarios；

  • discovering insight within large data collection，如不能手工绘制精密仪器；

3. When Not

  • A：若是基于简单规则构建的系统，则无需使用ML/DL；

  • B：当需解释性(explainability)，DL不适应；

  • DL模型通常有数百万，千万、亿、十亿，万亿的参数，如有一百万不同的事情发生，
    模型可能都很难理解三四个数字，如连续变调，更何况百万甚至数十亿数字；

  • C：当传统方法是更好的选择时，DL自然不是首选；

  • D：Error Unacceptable，DL的输出并非总是可预测predictable，
    错误在不可接受的情况下非常相似，DL模型是概率性Probabilistic；

  • E：，DL需大数据集Large Dataset；

6.1. 有监督

  • 有监督学习：当有数据data和标签label时，
    构建NN和ML算法来找出规则的方法(figure out rule)；

  • 有监督学习：将有大量数据，如输入大量原料和案例(example)；

  • 如辨别猫和狗：如我有一千张猫的图片，和一千张狗的照片，我知道那些是猫，
    那些是狗，然后将这些照片传递给ML算法来识别discern，即我们拥有数据，
    图片和标签(即每张照片是猫还是狗)，这就是有监督学习数据和标签；

6.2. 无监督和自监督

  • 无监督和自监督：只有数据本身，没有任何标签，即仅根据数据本身学习；

  • 那inherent representation，pattern in number，
    weight，feature是啥：即描述某一事物的一大堆属性名称；

  • 可使用自监督来找出猫和狗图片之间的基本模式pattern，但不一定知道两者之间的区别；

6.3. 迁移

  • 迁移学习是DL中的重要范式，采用一个模型从数据集学到的模式，并将其转移到另一个模型；

  • taking pattern A model learn from dataset and transferring to B model；

7. 强化

  • 强化学习理念：我们有某种环境(environment)和在此环境中采取行动(action)的代理人(agent)，
    我们会给代理人奖励和观察；我们会给代理人奖励(reward)和观察(observation)；

  • 如我们想教小狗在外面小便，会奖励它在外面小便的行为，而不奖励小便的行为；
