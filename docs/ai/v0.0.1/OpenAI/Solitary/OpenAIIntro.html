2. Key Concept

2.1. Text Generation Model

  • OpenAI文本生成模型通常称为GPT模型，如GPT-3.5，GPT-4，
    已经过训练来理解自然和形式语言understand natural and formal，

  • 像GPT-4类模型允许文本输出来响应其输入，输入也称提示prompt；
    设计(design)prompt本质上是如何编程(program)GPT-4等模型；

  • 通常通过提供指令(instructions)或范例(example)如何成功完成任务(task)；

  • GPT-4等模型可用于多种任务，如内容或代码生成，
    摘要summarization，对话conversation，创意写作等；

2.2. Assistant

  • Assistant指实体(refer to entity)，Assistant根据模型上下文窗口中嵌入的指令进行操作，
    assistant operate based on instruction embedded within model context window；

  • Assistant通常还可使用允许Assistant执行更复杂任务的工具，如运行代码或从文件中检索信息；

2.3. Embedding

  • Embedding是一段数据(如文本)的矢量表示，vector representation of data piece；

  • 旨在保留其内容和/或其含义的各方面，
    meant to preserve aspect of its content and/or its meaning；

  • 在某些方面相似的数据块往往比不相关的数据具有更紧密的嵌入；

  • data chunk are similar in some way will tend to have
    embedding that are closer together than unrelated data

  • OpenAI提供文本嵌入模型，将文本字符串作为输入并生成嵌入向量作为输出，

  • text embedding model take str as input
    and produce embedding vector as output；

  • Embedding对search搜索，clustering聚类，recommendation推荐，
    anomaly detection异常检测，classification分类等非常有用；

2.4. Token

  • 文本生成和嵌入模型以称为token的块的形式处理文本；

  • text generation and embedding model process text in chunk call token；

  • token表示(represent)常见的字符串序列character sequence，
    如tokenization被分解(decompose)为token和ization；

  • 而像the这样的短而常见的单词被表示为单个token；

  • 注：在句子(sentence)中，每个单词的第一个token通常以空格字符串开头，
    查看tokenizer tool，来测试特定字符串并查看它们如何转换成(translate into)token；

  • 根据粗略经验：1 token大约相当于4个字符character或英文文本的0.75个单词word；

  • 限制limitation：对文本生成模型，提示和生成的输出的总和不得超过模型的最大上下文长度；
    对嵌入模型embedding model(不输出token)，输入必须短于模型的最大上下文长度；
