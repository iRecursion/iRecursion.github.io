1. Intro

  • use GPT-4 to understand image

  • GPT-4 Turbo with Vision允许模型拍摄图像并回答有关图像的问题；

  • 从历史上看Historically，语言模型系统受到单一输入模式modality(文本)的限制；

  • 对许多用例来说，这限制GPT-4等模型的使用领域，此前该模型有时在Api中被称为
    GPT-4V或gpt-4-vision-preview，注：Assistant Api目前不支持图像输入；

2. Quick Start

  • 图像Image可通过两种主要方式提供给模型：
    传递图像的链接或直接在请求中传递base64编码的图像

  • 图像可在user，system和assistant message中传递，目前第一条系统消息中不支持图像；

from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
  	model="gpt-4-turbo",
  	messages=[{
      	"role": "user",
      	"content": [{"type": "text", "text": "What’s in this image?"},
        	{
          		"type": "image_url",
          		"image_url": {
            		"url": "https://upload.wikimedia.org/wikipedia/commons/thumb/d/dd/Gfp-wisconsin-madison-the-nature-boardwalk.jpg/2560px-Gfp-wisconsin-madison-the-nature-boardwalk.jpg",
          		},
        	},
      	],
	}],
  	max_tokens=300,
)

print(response.choices[0])

  • 该模型最擅长回答有关图像中存在的内容的一般问题，虽它确实理解图像中
    对象之间的关系，但尚未优化以回答有关图像中某些对象位置的详细问题；

  • 如可询问它汽车是什么颜色，或根据冰箱里的东西提出一些关于晚餐的想法，
    但若向它展示房间的图像并询问它椅子在哪里，它可能不会正确回答问题；

  • 当探索视觉理解可应用于那些用例时，务必牢记模型的局限性；

3. Upload Base64 Encode Img

  • 若本地有一个图像或一组图像，则可将它们以Base64编码格式传递给模型：

import base64
import requests

api_key = "YOUR_OPENAI_API_KEY"

def encode_image(image_path):
  with open(image_path, "rb") as image_file:
    return base64.b64encode(image_file.read()).decode('utf-8')

image_path = "path_to_your_image.jpg"
base64_image = encode_image(image_path)

headers = {
  	"Content-Type": "application/json",
  	"Authorization": f"Bearer {api_key}"
}

payload = {
  	"model": "gpt-4-turbo",
  	"messages": [{
      	"role": "user",
      	"content": [{
          	"type": "text",
          	"text": "What’s in this image?"
    	},{
          	"type": "image_url",
          	"image_url": {
            	"url": f"data:image/jpeg;base64,{base64_image}"
          	}
        }]
	}],
  	"max_tokens": 300
}

response = requests.post("https://api.openai.com/v1/chat/completions",headers=headers,json=payload)
print(response.json())

4. Multiple Image Input

  • Chat Completion Api能接收并处理采用Base64编码格式或图像Url
    形式的多个图像输入，该模型将处理每张图像并使用所有图像的信息来回答问题；

  • 此处模型显示同一图像的两个副本，且可独立回答有关两个或每个图像的问题；

from openai import OpenAI

client = OpenAI()
response = client.chat.completions.create(
  	model="gpt-4-turbo",
  	messages=[{
      	"role": "user",
      	"content": [{
          	"type": "text",
          	"text": "What are in these images? Is there any difference between them?",
    	},{
          	"type": "image_url",
          	"image_url": {
            	"url": "https://upload.wikimedia.org/wikipedia/commons/thumb/d/dd/Gfp-wisconsin-madison-the-nature-boardwalk.jpg/2560px-Gfp-wisconsin-madison-the-nature-boardwalk.jpg",
          	},
        },{
          	"type": "image_url",
          	"image_url": {
            	"url": "https://upload.wikimedia.org/wikipedia/commons/thumb/d/dd/Gfp-wisconsin-madison-the-nature-boardwalk.jpg/2560px-Gfp-wisconsin-madison-the-nature-boardwalk.jpg",
          	},
        },
      ],
	}],
  	max_tokens=300,
)
print(response.choices[0])

5. Fidelity Image Understanding

  • 通过控制细节参数(三个选项：low，high，auto)，可控制模型如何处理图像并生成其文本理解；

  • 默认模型将使用auto，该设置将查看图像输入尺寸并决定是否应使用low或high；

from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
  	model="gpt-4-turbo",
  	messages=[{
      	"role": "user",
      	"content": [{"type": "text", "text": "What’s in this image?"},
        	{
          		"type": "image_url",
          		"image_url": {
	            	"url": "https://upload.wikimedia.org/wikipedia/commons/thumb/d/dd/Gfp-wisconsin-madison-the-nature-boardwalk.jpg/2560px-Gfp-wisconsin-madison-the-nature-boardwalk.jpg",
	            	"detail": "high"
          		},
        	},
      	],
    }],
  	max_tokens=300,
)

print(response.choices[0].message.content)

5.1. low

  • 将启用低分辨率模式low res(resolution) mode；

  • 该模型将收到图像的低分辨率512px * 512px版本，并以65个token的预算表示该图像；

  • 使Api能返回更快的响应，并在不需高细节的用例中消耗更少的input token；

5.2. high

  • 将启高分辨率模式high res(resolution) mode；

  • 该模式首先允许模型查看低分辨率图像，然后根据输入图像大小将输入图像创建为512px
    正方形的详细裁剪crop，每个详细裁剪crop使用两倍的token预算，总计129个token；

6. Managing Image

  • 与Assistant Api不同，Chat Completion Api没状态，即必须自己管理传递给模型的
    消息(包括图像)，若想多次将相同的图像传递给模型，则每次向Api发出请求时都必须传递该图像；

  • 对长时间运行的对话，建议通过url而非Base64传递图像，
    还可通过提前将图像尺寸缩小到小于预期的最大尺寸来改善模型的延迟；

  • 对低分辨率模式，预计图像尺寸为512px * 512px，
    对高分辨率模式，图像的短边应小于768px，长边应小于2000px；

  • 图像经模型处理后，将从OpenAI服务器中删除且不会被保留，
    不使用通过OpenAI Api上传的数据来训练模型；

9. Faq

  • 目前不支持GPT-4来微调(fine-tune)image capability；

  • 不可用使用GPT-4生成图像，可使用dall-e-3生成图像，
    并使用gpt-4-turbo来理解图像(understand image)；

  • 支持上传的图像文件类型：png，jpg，jpeg，webp，gif；

  • 上传图像大小限制：每张图像最大20MB；

  • 用户不能删除上传的图像，在模型处理完图像后，会自动删除该图像；

  • 在token级别处理图像，处理的每个图像
    都会计入每分钟token(TPM：token per minute)的限制，

  • 带有Vision的GPT-4不可以理解图像元数据，模型不接受图像元数据；

  • 若图像不清楚ambiguous或不明确unclear，模型将尽力解释，但结果可能不太准确；

  • 好的经验法则：若普通人无法以低/高分辨率模式下使用的分辨率看到图像中的信息，那模型也看不到；
