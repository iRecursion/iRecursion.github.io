1. Intro

  • 将文本转换为逼真的语音，turn text into lifelike spoken audio；

  • 基于TTS(文本转语音text-to-speech)模型的语音端点speech endpoint；

  • 有6中内置语言，可叙述一篇博客文章，制作多语言的语音音频，用流式传输提供实时音频输出；

  • 注：使用政策要求向最终用户明确披露clear disclosure，
    他们听到的TTS语音是AI生成的，而非人类的声音；

2. Quick Start

  • 语音端点speech endpoint接受三个关键输入：

    • A：模型 model；

    • B：应转换为音频的文本 text should turn into audio；

    • C：用于音频生成的语音 voice used for audio generation；

from pathlib import Path
from openai import OpenAI

client = OpenAI()

speech_file_path = Path(__file__).parent / "speech.mp3"
response = client.audio.speech.create(
  	model="tts-1",
  	voice="alloy",
  	input="wonderful day for tts!"
)

response.stream_to_file(speech_file_path)

3. Audio Quality

  • 对实时应用，标准 tts-1 模型提供最低延迟latency，但质量低于 tts-1-hd 模型；

  • 因音频audio的生成方式，tts-1 在某些情况下可能会生成比 tts-1-hd 更静态的内容；

  • 在某些情况下，根据我们的收听设备listening device和个人individual
    person的不同，音频可能没明显的差异noticeable difference；

4. Voice Option

  • 尝试不同的声音experiment with different voice：
    alloy，echo，fable，onyx，nova，shimmer；

5. Output Format

  • mp3(默认)，opus，aac，flac，pcm；

5.1. Opus

  • 用于互联网流媒体和通信，低延迟；

  • for internet streaming and communication，low latency；

5.2. AAC

  • 用于数字音频压缩，YouTube，Android，iOS首选；

  • for digital audio compression，preferr by YouTube，Android，iOS；

5.3. FLAC

  • 用于无损音频压缩，受音频爱好者存档的青睐；

  • for lossless audio compression，
    favor by audio enthusiast for archiving

5.4. WAV

  • 未压缩的wav音频，适合低延迟应用，以避免解码开销；

  • uncompressed wav audio，suitable for low-latency
    application to avoid decoding overhead；

5.5. PCM

  • ：与wav类似，但包含24kHz的原始样本(16位有符号、低端)，没标头；

  • similar to wav but containing raw sample in 24kHz
    (16-bit signed，low-endian)，without header；

6. Streaming RealTime Audio

  • Speech Api使用块传输编码提供对实时音频流的支持，即可在生成完整文件前可访问音频；

  • real time audio streaming using chunk transfer encoding；

from openai import OpenAI

client = OpenAI()

response = client.audio.speech.create(
	model="tts-1",
    voice="alloy",
    input="hello world，streaming test!",
)

response.stream_to_file("streaming-tts.mp3")

7. Faq

  • 控制生成的音频的情感范围，control generated audio emotional range；

  • 没直接的机制来控制所生成的音频的情感输出，某些因素可能会影响输出音频，
    如大小写或语法，但对这些因素的内部测试产生不同的结果；
