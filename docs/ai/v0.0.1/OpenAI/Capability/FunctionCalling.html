1. Intro

  • 如何将大型语言模型连接到外部工具，
    connect large language model to external tool；

  • 在Api调用中，可描述函数(describe function)，
    并让模型智能的intelligently选择输出包含调用一或多个函数的参数的json对象；

  • Chat Completions Api不会调用该函数，
    相反模型会生成json，看使用它来调用代码中的函数；

  • 最新模型gpt-3.5-turbo-0125和gpt-4-turbo-preview经过训练，
    可检测何时应调用函数(取决于输入)并使用符合函数签名的json进行响应；

  • 此能力也带来潜在风险，建议在代表用户采取影响世界的行动
    (如发送右键，在线发布内容，购买等)之前构建用户确认流程；

2. Common Use Case

  • 函数调用使我们能更可靠地reliably从模型中获取结构化数据；

  • A：创建通过调用外部Api回答问题的助手，
    create assistant answer question by calling external Api：

  • 如定义send_email(to:string,body:string)或
    get_current_weather(location:string,unit:'celsius'|'fahrenheit')之类的函数；

  • B：将自然语言转换为Api调用：如转换"谁是我的主要(top)客户"：
    get_customers(min_revenue:int,created_before:string,limit:int)并调用我们的内部Api；

  • C：从文本中提取结构化数据：

  • 如定义名为extract_data(name:string,birthday:string)
    或sql_query(query:string)的函数；

  • 函数调用的基本步骤顺序：

  • A：使用用户查询和函数参数中定义的一组函数来调用模型，call model with
    user query and set of function define in function parameter；

  • B：模型可选择调用一或多个函数；那内容将是一个符合
    自定义schema的字符串化json对象(注：模型可能会产生参数)；

  • C：在代码中将字符串解析为json，并使用提供的参数(若存在)调用函数；

  • D：通过将函数响应作为新消息附加来再次调用模型，并让模型将结果汇总返回给用户；

3. Support Model

  • 支持函数调用的模型：gpt-4-turbo、gpt-4-turbo-2024-04-09、
    gpt-4-turbo-preview、gpt-4-0125-preview、
    gpt-4-1106-preview、gpt-4、gpt-4-0613、gpt-3.5-turbo、
    gpt-3.5-turbo-0125、gpt-3.5-turbo-1106和gpt-3.5-turbo-0613；

  • 支持并行函数调用的模型：gpt-4-turbo、gpt-4-turbo-2024-04-09、
    gpt-4-turbo-preview、gpt-4-0125-preview、
    gpt-4-1106-preview、gpt-3.5-turbo-0125和gpt-3.5-turbo-1106；

4. Function Calling Behavior

The default behavior for tool_choice is tool_choice: "auto". This lets the model decide whether to call functions and, if so, which functions to call.

We offer three ways to customize the default behavior depending on your use case:

To force the model to always call one or more functions, you can set tool_choice: "required". The model will then select which function(s) to call.
To force the model to call only one specific function, you can set tool_choice: {"type": "function", "function": {"name": "my_function"}}.
To disable function calling and force the model to only generate a user-facing message, you can set tool_choice: "none".

  • tool_choice的默认行为：tool_choice: "auto"，
    即让模型决定是否调用函数，若调用，则调用哪些函数；

  • 自定义默认行为：

    • 强制模型始终调用一或多个函数，
      可设置tool_choice: "required"，然后模型将选择要调用的函数；

    • 强制模型仅调用某特定函数，可做如下设置：

tool_choice:
	{
		"type": "function",
		"function": {
			"name": "my_function"
		}
	}

  • 要禁用函数调用并强制模型仅生成面向用户的消息，可设置 tool_choice: "none"；

5. Parallel Function Calling

  • 模型同时执行多个函数调用的能力，允许并行解决这些函数调用
    的效果effect和结果result，若函数需很长时间，这尤其有用；

  • 且可减少Api的往返次数，如模型可能会调用函数来同时获取3个不同位置的天气，
    这将导致在tool_calls数组中产生一条包含3个函数调用的消息，每个函数调用都有一个Id；

  • 要响应这些函数调用，请向对话中添加3条新消息，每条消息都包含
    一个函数调用的结果，并使用tool_call_id引用tool_calls中的Id；

6. Multiple Function Call

from openai import OpenAI
import json

client = OpenAI()

# dummy function hard code to return the same weather
# in production,this could be our backend api or external api
def get_current_weather(location, unit="fahrenheit"):

    """Get the current weather in a given location"""

    if "shanghai" in location.lower():
        return json.dumps({"location": "Shanghai", "temperature": "10", "unit": unit})
    elif "san francisco" in location.lower():
        return json.dumps({"location": "San Francisco", "temperature": "72", "unit": unit})
    elif "paris" in location.lower():
        return json.dumps({"location": "Paris", "temperature": "22", "unit": unit})
    else:
        return json.dumps({"location": location, "temperature": "unknown"})

def run_conversation():
    # Step A: send conversation and available functions to model
    messages = [{"role": "user", "content": "What's the weather like in San Francisco, Shanghai, and Paris?"}]
    tools = [
        {
            "type": "function",
            "function": {
                "name": "get_current_weather",
                "description": "Get the current weather in a given location",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "location": {
                            "type": "string",
                            "description": "The city and state, e.g. San Francisco, CA",
                        },
                        "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]},
                    },
                    "required": ["location"],
                },
            },
        }
    ]
    response = client.chat.completions.create(
        model="gpt-3.5-turbo-0125",
        messages=messages,
        tools=tools,
        tool_choice="auto",  # auto is default, but we'll be explicit
    )
    response_message = response.choices[0].message
    tool_calls = response_message.tool_calls

    # Step B: check if the model wanted to call a function
    if tool_calls:

        # Step C: call function
        # Note: the JSON response may not always be valid; be sure to handle errors
        available_functions = {
            "get_current_weather": get_current_weather,
        }  # only one function in this example, but you can have multiple
        messages.append(response_message)  # extend conversation with assistant's reply
        # Step D: send the info for each function call and function response to the model
        for tool_call in tool_calls:
            function_name = tool_call.function.name
            function_to_call = available_functions[function_name]
            function_args = json.loads(tool_call.function.arguments)
            function_response = function_to_call(
                location=function_args.get("location"),
                unit=function_args.get("unit"),
            )
            messages.append(
                {
                    "tool_call_id": tool_call.id,
                    "role": "tool",
                    "name": function_name,
                    "content": function_response,
                }
            )  # extend conversation with function response
        second_response = client.chat.completions.create(
            model="gpt-3.5-turbo-0125",
            messages=messages,
        )  # get a new response from the model where it can see the function response
        return second_response
print(run_conversation())

7. Token

  • 在底层，函数按照模型训练过的语法注入到系统消息中，即函数会根据模型的
    上下文限制进行计数count，
    并作为输入token进行计费，若遇到上下文限制，建议限制函数的数量或为函数参数提供的文档的长度；

  • 若已定义许多函数，还可使用微调来减少使用的token数量；
