1. Intro

  • turn audio into text，Audio Api基于最先进的
    (state-of-the-art)开源large-v2 Whisper model；

  • 提供两种语音到文本端点、转录和翻译，
    speech to text endpoint，transcription，translation；

  • A：将音频转录transcribe成音频所使用的任何语言；

  • B：将音频翻译并转录成英语，
    translate and transcribe audio into english；

  • 文件上传限制：25MB；

  • 支持文件类型：mp3，mp4，mpeg，mpga，m4a，wav，webm；

2. Quick Start

2.1. Transcription

  • 转录(transcription) Api：将想要转录的音频文件及音频
    转录所需的输出文件格式作为输入，目前支持多种输入和输出文件格式；

from openai import OpenAI
client = OpenAI()

audio_file= open("/path/to/file/audio.mp3", "rb")
transcription = client.audio.transcriptions.create(
  	model="whisper-1",
  	file=audio_file
)
print(transcription.text)

  • 默认响应类型为json，其中包含原始文本(raw text)：

{
  	"text": "Imagine the wildest idea that you've ever had,
  		and you're curious about how it might scale to
  		something that's a 100, a 1,000 times bigger."
}

  • Audio Api允许在请求中设置其它参数，如想将response_format设置为文本：

from openai import OpenAI
client = OpenAI()

audio_file = open("/path/to/file/speech.mp3", "rb")
transcription = client.audio.transcriptions.create(
  	model="whisper-1",
  	file=audio_file,
  	response_format="text"
)
print(transcription.text)

2.2. Translation

  • 翻译(translation) Api：将任何受支持语言的音频文件作为输入，并在必要时将音频转录transcribe为英语；

  • 这与/Transcriptions endpoint不同，因输出不是原始输入语言，而是翻译translate为英语文本；

from openai import OpenAI
client = OpenAI()

audio_file= open("/path/to/file/german.mp3", "rb")
translation = client.audio.translations.create(
  	model="whisper-1",
  	file=audio_file
)
print(translation.text)
hello,my name is elf and I come from China,where are you heading today?

We only support translation into English at this time.

3. Timestamp

  • 默认：Whisper Api将以文本形式输出所提供音频的转录本transcript；

  • timestamp_capsularities[]参数支持更加结构化和带时间戳的json输出格式，
    并在段segment、word level或两者上都带有时间戳；

  • 这可实现文字记录和视频编辑的单词级精度，从而可删除与各个单词相关的特定帧；

  • word-level precision for transcript and video edit，
    which allow for removal of specific frame tied to individual word；

from openai import OpenAI
client = OpenAI()

audio_file = open("speech.mp3", "rb")
transcript = client.audio.transcriptions.create(
  	file=audio_file,
  	model="whisper-1",
  	response_format="verbose_json",
  	timestamp_granularities=["word"]
)

print(transcript.words)

4. Longer Input

By default, the Whisper API only supports files that are less than 25 MB. If you have an audio file that is longer than that,
you will need to To get the best performance, we suggest that you avoid breaking the audio up mid-sentence as this may cause some context to be lost.
* 默认Whisper Api仅支持小于25MB的文件；

  • 若音频文件比该长度长，则需将其分成25MB或更少的块，或使用压缩音频格式，
    break it up into chunk of 25 MB；

  • 为获得最佳性能，建议避免在句子中间中断音频，因这可能会导致丢失一些上下文

  • 处理此问题的解决方法如：使用pydub包来分割音频；
    https://github.com/jiaaro/pydub

from pydub import AudioSegment

song = AudioSegment.from_mp3("good_morning.mp3")

# PyDub handle time in millisecond
ten_minutes = 10 * 60 * 1000

first_10_minutes = song[:ten_minutes]

first_10_minutes.export("good_morning_10.mp3", format="mp3")

5. Prompting

  • 可使用提示prompt来提高Whisper Api生成的transcript的质量；

  • 该模型尝试匹配提示的样式prompt style，
    故若提示prompt也这样做，则更有可能使用大写和标点符号；
    more likely to use capitalization and punctuation

  • 但当前的提示系统比我们的其它语言模型受到更多限制，且仅提供对生成的音频的有限控制；

  • 以下是提示如何在不同场景下提供帮助：

  • A：提示对纠正模型在音频中可能错误识别的特定单词或首字母缩略词非常有帮助，如以下提示
    改进单词DALL·E和GPT-3的转录transcription，这些词之前写为GDP 3和DALI；

  • correcting specific word or acronym model may misrecognize in audio；

  • B：要保留分割为多个片段的文件的上下文，可使用前一个片段的转录本来提示模型，
    这将使转录transcript更准确accurate，因模型将用之前音频中的相关信息；

  • to preserve context of file split into segment，
    prompt model with transcript of preceding segment；

  • 该模型将仅考虑提示的最后224个标记(final 224 token)，
    并忽略之前的任何内容ignore anything earlier；

  • 对多语言输入，Whisper用自定义标记器tokenizer，对仅英语输入，
    用标准GPT-2分词器，这两种分词器都可通过开源Whisper Python包访问；

  • https://github.com/openai/whisper/blob/main/whisper/tokenizer.py#L361

  • C：有时模型可能会跳过记录中的标点符号，
    skip punctuation in transcript；

  • 可通过使用包含标点符号的简单提示来规避此类情况：
    "您好，欢迎来到我的讲座"，"hello,welcome to my lecture"；

  • D：该模型还可能会遗漏音频中常见的填充词，
    leave out common filler word in audio；

  • 若想在transcript中保留填充词，可使用包含它们的提示：
    use prompt contain them to keep filler word in our transcript；

  • "Umm,let me think like,hmm…​ Okay,here’s what I’m,like,thinking."，
    "嗯，让我这样想，嗯…​…​好吧，这就是我的想法"；

  • E：有些语言可用不同的方式书写，如简体中文或繁体中文，默认，模型可能并不总是使用我们
    想要的transcript写作风格，可通过使用我们喜欢的写作风格writing style的提示来改进；

6. Improving Reliability

  • 使用Whisper棉铃挑战：如模型通常无法识别不常见的单词或首字母缩略词；
    recognize uncommon word or acronym

  • 可通过提高Whisper可靠性(reliability)不同技术来解决此问题；

6.1. Using Prompt Parameter

The first method involves using the s.
* 使用可选的提示参数来传递正确拼写的字典，optional prompt parameter
to pass dictionary of correct spelling；

  • 因Whisper没使用指令跟踪技术instruction-following technique训练，
    故其运行方式更像是基本GPT模型；

  • 注：Whisper仅考虑提示的前244个标记，considere first 244 token of prompt；

  • 虽会提高可靠性，但该技术仅限于244个字符character，
    故SKU列表需相对较小，才能成为可扩展的scalable解决方案；

from openai import OpenAI
client = OpenAI()

audio_file = open("/path/to/file/speech.mp3", "rb")
transcription = client.audio.transcriptions.create(
  	model="whisper-1",
  	file=audio_file,
  	response_format="text",
  	prompt="ZyntriQix,Digique Plus,CynapseFive,VortiQore V8,
  		EchoNix Array,OrbitalLink Seven, DigiFractal Matrix,
  		PULSE, RAPT, B.R.I.C.K., Q.U.A.R.T.Z., F.L.I.N.T"
)
print(transcription.text)

6.2. Post-processing with GPT-4

  • 使用GPT-4或GPT-3.5-Turbo的post-processing step；

  • 首先通过system_prompt变量提供GPT-4的指令instruction，类提示参数所做；

  • 若在自己的音频文件上尝试此操作，会发现GPT-4设法
    纠正记录transcript中的许多拼写错误misspelling；

  • 因其更大的上下文窗口，该方法可能比使用Whisper的提示参数更具可扩展性，
    且更可靠reliable，因GPT-4可以以Whisper无法实现的方式进行
    指示instruct和引导guide，因为缺乏后续指令lack of instruction following；

system_prompt =
	"You are a helpful assistant for the company ZyntriQix.
	Your task is to correct any spelling discrepancies in the transcribed text.
	Make sure that the names of the following products are spelled correctly:
	ZyntriQix, Digique Plus, CynapseFive, VortiQore V8, EchoNix Array,
	OrbitalLink Seven, DigiFractal Matrix, PULSE, RAPT, B.R.I.C.K., Q.U.A.R.T.Z.,
	F.L.I.N.T. Only add necessary punctuation such as periods, commas,
	and capitalization, and use only the context provided."

def generate_corrected_transcript(temperature,system_prompt,audio_file):
