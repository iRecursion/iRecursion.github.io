1. Introduction

  • 在向量数据库中，向量以机器能理解的格式表示复杂的数据，如文本，图像，声音等，
    它将定性qualitative属性转换为定量quantitative特征，可用数学运算轻松比较；

  • 如衣服的定性属性：尺寸：L，颜色：白色，袖子：长，面料：棉等；都可成为整个向量的值；

  • 创建向量，也称嵌入(Embeddings)，Embeddings是捕获其语义
    semantic meaning的数据点data point的向量表示；

  • 具有大量属性的数据，随维数的陡增，向量空间体积也增加，但管理和搜索成本高昂，
    数据点之间的距离变得不那么有意义，搜索相似点的计算复杂性变得昂贵，
    以至于可能不合理，可使用降维技术，如特征选择，哈希，主成分分析PCA；

  • Distance Metric是确定向量空间中两点之间"distance"的数学函数，
    不同的Distance Metric可以表示数据的不同方面之间的关系；

Type Memo

Euclidean Distance

对向量的大小magnitude敏感

Manhattan Distance

适于图像检索和财务分析

Cosine Distance

单词频率不如实际含义重要的情况下，很适合评估文本相似性

Jaccard Similarity

根据购买历史查找相似的客户

  • 找相似可通过KNN和ANN等算法实现，每种算法各有优缺点，这些算法
    也是数据库引擎的一部分，与任何数据库一样，索引是提高执行时间的关键；

  • 向量数据库，查询不同于传统关系数据库，执行相似性similarity搜索，
    而非精确匹配，当给定向量作为查询时，向量库会返回于查询向量相似的向量；

  • 向量数据库用于将数据库与AI模型集成，首先将数据导入向量数据库，其次当用户查询发送到AI模型时，
    首先检索一组类似文档，然后这些文档作为用户问题的上下文，与用户的查询一起发送到AI模型，
    这种技术称为检索增强生成RAG(Retrieval Augmented Generation)

2. Api Intro

  • Spring AI提供抽象Api，通过VectorStore接口与向量数据库交互；

public interface VectorStore {

    void add(List<Document> documentList);

    Optional<Boolean> delete(List<String> idList);

    List<Document> similaritySearch(String query);

    List<Document> similaritySearch(SearchRequest request);
}

public class SearchRequest {

	private int topK = 4;
	public final String query;
	private Filter.Expression filterExpression;

		this.query = query;

		return new SearchRequest(query);
	}

}

  • 要将数据插入向量数据库，请将其封装在Document对象中，Document类封装数据源
    (如Pdf或Word文档)中的内容，并包括以字符串表示的文本，它还包含键值对形式的元数据，包括文件名等详情；

  • 插入向量数据库后，文本内容将使用嵌入模型转换为数字(numerical)数组或List＜Double＞，
    称为矢量嵌入(vector embeddings)，嵌入模型，如Word2Vec、GLoVE和BERT，
    或OpenAI的text-Embedding-ada-002，用于将单词、句子或段落转换为这些矢量嵌入；

  • https://en.wikipedia.org/wiki/Word2vec

  • https://en.wikipedia.org/wiki/BERT_(language_model)

  • https://en.wikipedia.org/wiki/GloVe_(machine_learning)

  • 向量数据库的作用：存储store和促进facilitate对这些嵌入的相似性搜索，
    它本身不会生成嵌入，为创建向量嵌入，应该使用EmbeddingClient；

  • similaritySearch()允许检索与给定查询字符串相似的文档，可使用以下参数对这些方法微调：
    fine-tuned by using the following parameters:

    • k：整数，指定要返回的类似文档的最大数量，通常称为"top K"搜索或KNN(K个最近邻居)；

    • threshold：范围从0到1的两倍值，其中接近1的值表示相似性更高，
      默认如若将阈值设置为0.75，则只返回相似度高于该值的文档；

    • Filter.Expression：传递fluent DSL表达式的类，
      该表达式类似于SQL中where子句，但仅适用于文档的元数据键值对；

    • filterExpression：基于Antlr4的external Dsl，接受作为字符串的
      filter expression，如对country、year和isActive等元数据键，可使用如下表达式：
      country == 'UK' && year >= 2020 && isActive == true

4. Usage

  • 要计算向量数据库的嵌入embeddings，需选择一个与所使用的高级AI模型匹配的嵌入模型；
    如对ChatGPT，用OpenAiEmbeddingClient和模型名称text-embedding-ada-002；

  • boot starter对OpenAI的自动配置使EmbeddingClient的实现可在Spring
    应用上下文中用于依赖项注入，将数据加载到向量存储的一般用法是在类似批处理的工作中执行的，
    方法是首先将数据加载至Spring AI的Document类，然后调用save()；

  • 给定一个对源文件的String引用，该源文件表示Json文件，包含要加载到向量数据库中的数据，
    用Spring AI的JsonReader加载JSON中的特定字段，该字段将它们拆分为小块piece，
    然后将piece传递给向量存储的实现，VectorStore实现计算嵌入并将Json和嵌入存储在向量数据库中；

@Autowired

5. Metadata Filter

  • 用于查询结果的各种过滤器

5.1. Filter String

  • 可将类似Sql的过滤器表达式作为字符串传递给重载similaritySearch()之一；

  • "country == 'BG'"

  • "genre == 'drama' && year >= 2020"

  • "genre in ['comedy', 'documentary', 'drama']"

5.2. Filter.Expression

  • 可创建Filter的实例，带有FilterExpressionBuilder
    的表达式，该表达式公开流畅的(fluent)API：

FilterExpressionBuilder b = new FilterExpressionBuilder();
Expression expression = b.eq("country", "BG").build();

  • 可使用运算符构建复杂的表达式：
