1. Introduction

  • Neo4jVectorStore存储document embedding并执行similarity search，

  • Neo4j是开源图数据库，是完全事务性数据库(ACID)，由节点node
    和关系relationship组成，允许对复杂数据保持高查询性能；

  • Neo4j的向量搜索允许用户从大数据集中查询vector embedding，
    embedding是数据对象的数字表示，如文本、图像、音频或文档；

  • embedding可存储在Node属性上，用db.index.vector.queryNodes()查询，
    索引由Lucene支持，用Hierarchical Navigable Small World Graph(HNSW)
    对向量字段执行k近似最近邻(k-ANN)查询(k approximate nearest neighbor)；

4. Metadata Filtering

  • text expression language方式：

var textExpression = "author in ['john', 'jill'] && 'article_type' == 'blog'";
vectorStore.similaritySearch(SearchRequest.defaults()
    .withQuery("The World")
    .withTopK(TOP_K)
    .withSimilarityThreshold(SIMILARITY_THRESHOLD)
    .withFilterExpression(textExpression));

  • programmatically Filter.Expression Dsl方式：

var builder = new FilterExpressionBuilder();
vectorStore.similaritySearch(SearchRequest.defaults()
    .withQuery("The World")
    .withTopK(TOP_K)
    .withSimilarityThreshold(SIMILARITY_THRESHOLD)
    .withFilterExpression(b.and(
		builder.in("john", "jill"),
		builder.eq("article_type", "blog")).build()));

  • 这些(portable) filter expression可自动转换为
    proprietary Neo4j WHERE filter expression：

  • portable filter expression：
    author in ['john', 'jill'] && 'article_type' == 'blog'

  • proprietary Neo4j filter format：
    node.metadata.author IN ["john","jill"] AND node.metadata.'article_type' = "blog"
