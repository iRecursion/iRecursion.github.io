1. Class Diagram

2. ModelClient

  • ModelClient接口为调用AI模型提供通用的API，
    抽象发送请求和接收响应来处理与各类AI模型的的交互；

public interface ModelClient<TReq extends ModelRequest<?>,
	TRes extends ModelResponse<?>> {

	/**
	 * execute method call to AI model
	 * @param TReq AI model request
	 * @return TRes AI model response
	 */
	TRes call(TReq request);

}

3. StreamingModelClient

  • StreamingModelClient接口提供通用的API，
    调用具有流式响应的AI模型，抽象发送请求和接收流式响应；

public interface StreamingModelClient<TReq extends ModelRequest<?>,
	TResChunk extends ModelResponse<?>> {

	/**
	 * execute method call to AI model
	 * @param TReq AI model request
	 * @return TRes AI model streaming response
	 */
	Flux<TResChunk> stream(TReq request);

}

4. ModelRequest

  • 对AI模型的请求的接口，封装与AI模型交互所需的必要信息，
    包括instruction or input (of generic type T)和其他模型选项，
    提供向AI模型发送请求的标准化方式，确保所有必要的细节都包括在内，且可以轻松管理；

public interface ModelRequest<T> {

	//AI model所需的instruction or input
	T getInstructions();

	 //AI模型交互的可自定义选项customizable option
	ModelOptions getOptions();

}

5. ModelOptions

public interface ModelOptions {}

  • AI模型交互的可自定义选项，该标记marker允许指定各种设置和参数，
    设置和参数可影响AI模型的行为和输出，以在不同AI场景提供灵活性和适应性，
    确保AI模型可根据特定需求requirement微调fine-tuning；

6. ModelResponse

  • 从AI模型接收到的响应的接口，提供访问AI模型生成的主要结果或结果列表及响应元数据，
    封装和管理AI模型输出的标准化方式，确保生成的信息易于检索和处理；

public interface ModelResponse<T extends ModelResult<?>> {

	//查询AI模型生成的结果
	T getResult();

	//查询AI模型生成的结果列表
	List<T> getResults();

	//查询与AI模型响应相关联的元数据response metadata
	ResponseMetadata getMetadata();

}

7. ModelResult

  • 提供访问AI模型的主要输出及与该结果相关联的元数据，旨在提供标准化standardize和
    全面的comprehensive方式来处理handle和解释interpret AI模型产生的输出；

public interface ModelResult<T> {

	//检索AI模型生成的输出
	T getOutput();

	//检索与AI模型的结果相关联的元数据
	ResultMetadata getMetadata();

}
