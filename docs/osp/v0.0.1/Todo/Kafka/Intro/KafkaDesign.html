2. Persistence

  • Kafka依赖于文件系统来存储和缓存消息，磁盘可很快或很慢，取决于使用方式，
    如7200转某磁盘，线性写入的性能约600MB/s，但随机写入的性能仅为约100k/s，
    相差6000倍，经OS等各种优化，顺序磁盘访问可能比随机内存访问更快；

  • 为弥补这种性能差异，现代OS用主内存进行磁盘缓存，当内存被回收时，OS将所有可用内存
    转移到磁盘缓存，而几乎不会会影响性能，所有磁盘读取和写入都将通过此统一的缓存；

  • 另构建在Jvm上，对象的内存开销很高，通常会使存储的数据大小加倍，甚至更糟，
    随堆内数据增加，垃圾收集变得越来越麻烦和缓慢；

  • 故用文件系统和依赖页面缓存优于维护内存中的缓存或其他结构，通过自动访问所有可用内存，
    至少将可用缓存增加一倍，并可能通过存储紧凑的字节结构而非单个对象，再次增加一倍，
    即将在32G的机器获得高达28~30GB的缓存，而不会受到GC处罚；

  • 另即使重新启动服务，该缓存也会保持温暖，而进程中缓存将需在内存中重建(10G缓存约需10分钟)，
    否则它将需从完全冷的缓存开始，这可能意味糟糕的初始性能，也大大简化代码，因用于维护缓存和
    文件系统间一致性的所有逻辑现在都在OS中，这往往比一次性的进程内尝试更有效、更正确；

  • 若磁盘使用倾向于线性读取，那预读实际上是在每次读取磁盘时用有用的数据预先填充此缓存，
    当空间用完时，不是在内存中维护尽可能多的内存，并在恐慌中将其全部清空到文件系统，
    而是将其颠倒过来，所有数据都会立即写入文件系统上的持久日志，而不必刷新到磁盘，
    即这只是意味着它被转移到内核的页面缓存中；
