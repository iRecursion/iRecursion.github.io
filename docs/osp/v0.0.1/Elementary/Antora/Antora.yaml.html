1. name

  • name：必选，组件名称：name和version联合标识组件；

  • 接受字母、数字、下划线(_)、连字符(-)和句点(.)，建议字母小写；

  • 组件名称值不能为空，不能包含空格、正斜杠(/)或Html特殊字符(&、<或>)；

2. start_page

  • start_page：可选，指定组件版本的主页；

  • 默认使用ROOT模块中的index.adoc；

  • start_page: operation:Entry.adoc；

3. version

  • 必选，version可从playbook继承，也可自定义；

  • 定义没版本的组件，使用波浪号，是null的简写；

  • 值不能包含空格，正斜杠(/)或Html特殊字符(&，<或>)；

  • 值可包含字母(应小写)，数字，句点(.)，下划线(_)，连字符(-)

4. nav

  • nav：可选：接受导航文件列表

5. title

  • title：可选，仅在引用UI的组件名称，接受空格、大写字母和大量字符

6. asciidoc.attributes

  • 可选，属性默认硬设置(hard set)：属性没以@结尾修饰，
    且属性被分配不是为取消设置而保留的值(即false或~)，

  • 属性在playbook是hard set或hard unset，
    那在antora.yml中声明的属性不能覆盖相同的属性

  • 只有当站点属性为soft set或unset时，组件属性才能覆盖该站点属性

  • 当声明组件版本属性时，可根据具体情况控制页面是否可reset或unset属性。
    假定属性是soft set或unset，或之前在playbook未定义，

  • 那hard set或hard unset的组件版本属性就不能被页面更改，
    但soft set和unset的组件版本属性可从页面覆盖。

name: management
title: Management
version: 'v0.0.1'

asciidoc:
  attributes:
    attribute-missing: 'warn'

    table-caption: Data Set@
    hide-uri-scheme: '@'

    table-caption: ~
    hide-uri-scheme: ~

    table-caption: false
    hide-uri-scheme: false

nav:
- modules/Process/nav.adoc
- modules/Changeable/nav.adoc
- modules/Principle/nav.adoc
Type Memo

Hard Unset

无尾随@修饰符，硬设置属性可防止页面更改或取消设置该属性

Soft Unset

带有尾部@修饰符，软设置属性允许页面更改或取消设置该属性

Hard Unset

使用波浪号符号(~)，硬取消设置属性会阻止页面设置该属性

Soft Unset

使用false，软取消设置属性允许页面设置该属性
