1. Intro

  • Doxygen是基于命令行的应用程序，使用doxygen --help查看帮助的简要描述；

  • 所有选项都由前导字符(leading character：- )组成，
    一个字符和一或多个参数组成，具体视选项而定；

  • 可使用特殊的文档块来文档化源代码：Special Comment Block：
    https://www.doxygen.nl/manual/docblocks.html#specialblock

  • 配置文件从旧版本升级到当前版本：任何新选项都将具有默认值，
    注：原始配置文件中添加的注释将丢失；

2. Generate Manual

Entry Command

生成配置文件

doxygen -g <config_file>

生成文档

doxygen <config_file>

升级配置文件版本

doxygen -u <config_file>

3. Fine Tuning Output

  • 可微调输出的外观，Doxygen允许生成默认的style sheet，
    header，footer，之后(afterward)可编辑；

  • 自定义Header时，有责任正确包含Doxygen所需的任何script和style sheet，
    这取决于配置选项，在升级到新的Doxygen版本时可能发生变化；

  • 若不希望配置文件中每个item都有文档，则可使用 -s 选项，
    可与 -u 选项结合使用，以添加或删除现有配置文件中的文档；

  • 要获得最小配置，可使用 -x 或 -x_noenv选项仅显示与默认Doxygen配置文件的查阅；

3.1. Html Output

  • A：Html输出：生成默认的Http Header，Html Footer，Html Style Sheet；

  • doxygen -w html header.html footer.html stylesheet.css <config_file>

  • config_file可选，省略时，Doxygen将搜索名为
    Doxyfile的文件并process，若没找到，则使用默认设置；

  • https://www.doxygen.nl/manual/config.html#cfg_html_header

  • https://www.doxygen.nl/manual/config.html#cfg_html_footer

  • https://www.doxygen.nl/manual/config.html#cfg_html_stylesheet

3.2. LaTeX Output

  • 可使用命令生成refman.tex的LATEX_HEADER和LATEX_FOOTER，
    及Header包含的style sheet，通常称为通常为doxygen.sty；

  • doxygen -w latex header.tex footer.tex doxygen.sty <config_file>

  • https://www.doxygen.nl/manual/config.html#cfg_latex_header

  • https://www.doxygen.nl/manual/config.html#cfg_latex_footer

  • 若需非默认选项(如使用extra LaTeX package)，需制作配置文件，
    并正确设置这些选项，然后在生成的文件后指定这些配置文件(先备份配置文件)；

3.3. Rtf Output

  • 可使用命令生成默认的style sheet文件：RTF_STYLESHEET_FILE：

  • doxygen -w rtf rtfstyle.cfg

  • https://www.doxygen.nl/manual/config.html#cfg_rtf_stylesheet_file
