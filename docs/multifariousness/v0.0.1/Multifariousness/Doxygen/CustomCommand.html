1. Intro

  • Doxygen提供大量(Special，Xml，Html) Command，增强或构造
    注释块内的文档，可通过配置文件中ALIASES标签，来实现自定义新命令；

2. Simple Alias

name=value

ALIASES += sideeffect="\par Side Effects:^^"

  • 允许将命令 \sideeffect(或@sideeffect)放入文档中，
    这将导致用户定义的段落标题为Side Effects:

  • 注：不能将 \n放在alias的值部分来插入换行符(在结果输出中)，
    可将^^放在alias的值部分来插入换行符，就像原始文件中有物理换行符；

  • 注：当alias的值部分需{或}或非默认分隔符时，
    必须使用反斜杠(\)转义，这可能导致与命令\{ 和 \}发生冲突；

  • 故建议使用版本@{和@}或使用双转义(\\{ 和 \\})；
    另：若愿意，可重新定义现有特殊命令；

  • 某些命令，如\xrefitem，旨在与alias结合使用；

3. Alias With Argument

  • Alias可有一或多个参数，在Alias定义中，需在花括号(curly brace)间指定参数的数量；

  • 在定义值的部分，可放置 \x marker，其中 x 表示从 1 开始的参数编号；

  • A：单参数定义Alias：

ALIASES += l{1}="\ref \1"

  • B：在注释块(comment block)使用

/** See \l{SomeClass} for more information. */

/** See \ref SomeClass for more information. */

  • C：可使用具有多个参数的版本重载Alias：

ALIASES += l{1}="\ref \1"
ALIASES += l{2}="\ref \1 \"\2\""

  • 注：Alias定义中的引号必须用反斜杠转义

/** See \l{SomeClass,Some Text} for more information. */
/** See \ref SomeClass "Some Text" for more information. */

  • D：其中带有单个参数的命令仍可像之前所示那样工作
    Alias也可用其他Alias来表示，如新命令 \reminder
    可通过中间 \xreflist 命令表示为 \xrefitem，如：

ALIASES += xreflist{3}="\xrefitem \1 \"\2\" \"\3\" "
ALIASES += reminder="\xreflist{reminders,Reminder,Reminders}"

  • E：注：若对具有多个参数的Alias，用逗号作分隔符，
    若想在命令中放逗号，则需用反斜杠将其转义，即：

\l{SomeClass,Some text\, with an escaped comma}

  • 给出上例中 \l 的别名定义，默认：Alias中参数的分隔符是逗号
    但对带有大量逗号的参数(如函数定义的模板)，转义每个逗号可能很麻烦，

  • 解决方案：可在参数计数后直接指定不同的分隔符，
    如要使用分号作为分隔符，可按如下方式定义命令

ALIASES += xreflist{3;}="\xrefitem \1 \"\2\" \"\3\" "
ALIASES += reminder="\xreflist{reminders;Reminder;Reminders}"

  • F：注：也允许使用多字符分隔符，即可使用双管道符号作为分隔符来编写相同的示例：

ALIASES += xreflist{3||}="\xrefitem \1 \"\2\" \"\3\" "
ALIASES += reminder="\xreflist{reminders||Reminder||Reminders}"

  • G：允许使用以下字符来创建分隔符：

!#$%&,.?|;:'+=~`/

  • H：对每个命令和参数数量，可使用不同的分隔符，但不建议为同一命令选择不同的分隔符，
    因这可能会导致对要使用哪个命令定义的歧义产生歧义，Doxygen通过选择
    与最多参数匹配的命令来解决这种歧义，请考虑以下相当牵强的例子：

ALIASES += v{2+}="Choose 2: '\1' and '\2'"
ALIASES += v{3;}="Choose 3: '\1', '\2', and '\3'"

  • 然后：

- \v{One+Two}
- \v{One;Two;Three}
- \v{One+Two;Three;Four}

  • 产生：

Choose 2: 'One' and 'Two'
Choose 3: 'One', 'Two', and 'Three'
Choose 3: 'One+Two', 'Three', and 'Four'

  • 对最后一个命令，v的两个定义都匹配，
    但选择具有3个参数的定义，因它匹配更多参数；

4. Nesting Custom Command

  • 可将命令用作别名的参数，包括使用别名定义的命令

ALIASES += Bold{1}="<b>\1</b>"
ALIASES += Emph{1}="<em>\1</em>"

  • inside comment block：

/** This is a \Bold{bold \Emph{and} Emphasized} text fragment. */

  • expand to：

/** This is a <b>bold <em>and</em> Emphasized</b> text fragment. */
