1. Intro

  • 可执行文件doxygen是解析源代码并生成文档的主程序；

  • 另也可可选的使用图形前端：doxywizard，用于编辑
    Doxygen使用的配置文件，并在图形环境中运行Doxygen；

  • https://www.doxygen.nl/manual/doxygen_usage.html
    https://www.doxygen.nl/manual/doxywizard_usage.html

2. Checking Doxygen Support

  • 首先确保语言是否被Doxygen识别，默认支持Idl，Java，Python，Vhdl等；

  • 可配置某些文件类型扩展名以使用某些解析器；也可用预处理器程序来支持完全不同的语言；

  • https://www.doxygen.nl/manual/config.html#cfg_extension_mapping
    https://www.doxygen.nl/helpers.html

3. Creating Configuration

  • Doxygen用配置文件来确定其所有设置，每个项目都应有独立的配置文件，
    项目可由单个源文件组成，也可递归扫描整个源树；创建模板配置文件：

 
doxygen -g <config-file>

  • config-file默认为Doxyfile，若已存在，则Doxyfile
    将在生成配置模板前将其重命名为<config file>.bak；

  • 若使用 -(减号)作为文件名，则Doxygen将尝试从
    标准输入(stdin)读取配置文件，这对脚本编写很有用；

  • 配置文件格式类似于Makefile，由多个assignment(tag)组成；

 
LANG = Java Python CangJie

HOBBY = Reading，Translating

  • 若有由源目录或树组成的大项目，应将根目录分配给INPUT tag，
    并将一或多个文件模式添加到FILE_PATTERNS tag(如： *.cj *.java)

  • 只有与其中一种模式匹配的文件才被解析
    (若省略模式，Doxygen支持的文件类型将使用典型模式列表)；

  • 对源树的递归解析，必须将recursive标记设置为YES，可用EXCLUDE和
    EXCLUDE_PATTERNS tag，如省略源代码树中所有测试目录，可用：

 
EXCLUDE_PATTERNS = */test/*

  • Doxygen根据文件扩展名以确定如何解析文件；

  • 任何未解析的扩展都可通过将其添加到FILE_PATTERNS，
    并设置适当的extension_MAPPING；

  • 将EXTRACT_ALL tag设置为YES，Doxygen会认为源码中
    所有内容都被文档化，而不会生成关于被被文档成员的结果警告；

  • consequence warnings about undocument member

    在源文件中定义交叉引用(cross-reference)有用；

  • 若将SOURCE_BROWSER设置为YES，Doxygen将生成此类交叉引用；

  • 若INLINE_SOURCES设置为YES，
    可将源代码直接包含在文档中，这对代码审查很方便；

4. Running Doxygen

doxygen <config-file>

  • 按设置，Doxygen将在目录中创建html，rtf，latex，xml，
    man和/或docbook目录，这些目录包含其生成相应格式的文档；

  • OUTPUT_DIRECTORY可修改输出写入的根目录；

  • HTML_OUTPUT，RTF_OUTPUT，LATEX_OUTPUT，XML_OUTPUT，
    MAN_OUTPUT，DOCBOOK_OUTPUT可修改特定格式的输出目录；

  • 若输出目录不存在，doxygen将尝试创建(但不会像mkdir -p递归创建整个路径)；

  • https://www.doxygen.nl/manual/config.html#cfg_output_directory

  • https://www.doxygen.nl/manual/config.html#cfg_html_output

  • https://www.doxygen.nl/manual/config.html#cfg_rtf_output

  • https://www.doxygen.nl/manual/config.html#cfg_latex_output

  • https://www.doxygen.nl/manual/config.html#cfg_xml_output

  • https://www.doxygen.nl/manual/config.html#cfg_man_output

  • https://www.doxygen.nl/manual/config.html#cfg_generate_treeview

4.2. LaTeX Output

  • 生成的LaTeX文档须首先由LaTeX编译器(MikTex)编译，
    doxygen将Makefile写入latex目录(Win平台还会生成make.bat)；

  • Makefile内容和目标取决于USE_PDFLATEX设置，若禁用(设置为NO)，
    则在latex目录中键入make会生成refman.dvi文件，用xdvi查看此文件，
    或输入make ps(需dvips)将其转换为PostScript文件refman.ps；

  • 要将2个page放在一个物理页面，请改用make ps_2on1，
    生成的PostScript文件可发送到PostScript打印机；

  • 若没PostScript打印机，可尝试用ghostscript
    将PostScript转换为自己打印机可以理解的内容；

  • 若已安装ghostscript解释器，可转换为pdf；
    只需输入make pdf或make pdf_2on1；

  • 要获得最佳pdf输出效果，应将PDF_HYPERLINKS和USE_PDFLATEX
    设置为YES，此情况下，Makefile将仅包含直接构建refman.pdf的目标；

  • https://www.doxygen.nl/manual/config.html#cfg_use_pdflatex

  • https://www.doxygen.nl/manual/config.html#cfg_pdf_hyperlinks

  • https://www.doxygen.nl/manual/config.html#cfg_use_pdflatex

4.3. Rtf Output

  • Doxygen将rtf输出合并到refman.rtf，此文件已针对导入Word进行优化；

  • 某些信息用字段编码，要显示实际值，需全选(Edit - select all)，
    然后切换字段(toggle field，右键单击并从下拉菜单中选择选项)；

4.4. Xml Output

  • Xml输出由Doxygen手机的信息的结构化"dump"组成；

  • 每个复合文件(class/namespace/file/…​)
    都有自己的xml文件，还有一个名为index.xml的索引文件；

  • 还会生成Combine.xslt，可用于将所有xml合并为一个文件；

  • Doxygen还会生成两个Xml Schema：
    index.xsd(用于索引)和compound.xsd(用于复合)；

  • schema描述可能的元素，属性及结构，即描述xml的语法，可用于验证或引导xslt脚本；

  • 在addon/doxmlparser目录中，可找到解析器库，
    以增量方式读取Doxygen生成的xml输出；

  • 有关库的接口，请参阅addon/doxmlparser/doxmparser/index.py
    和addon/doxmlparser/doxmlparser/compound.py；

4.5. Man Page Output

  • 可用man程序查看生成的手册页，需确保man目录位于man路径中(请参阅MANPATH环境变量)，
    man directory in man path (MANPATH environment variable)；

  • 注：man page格式的功能存在限制，如class diagram，
    cross reference and formula将会丢失；

5. Documenting Source

  • 若EXTRACT_ALL设置为NO(默认值)，则Doxygen将仅为已记录的实体(document entity)
    生成文档，如何文档化这些内容，对member，class，namespace，基本有两个选项；

    • A：在member，class，namespace的声明或定义前面放置一个特殊文档块，
      对member，class，namespace，也可将文档直接放置在成员之后；

    • Special Comment Block

    • B：将特殊文档块放在其他地方(另一个文件或其他位置)，并在文档块中放置结构命令，
      结构命令将文档块链接到可文档化的某个实体(如member，class，namespace)；

    • Documentation At Other Place

  • 选择A的优点：无需重复实体名词，entity name；

  • 只能用第二种选择来文档化文件，因没办法将文档块放在文件前面，当然文件成员(function，
    variable，typedef，define)无需显示结构命令，只需在它们前或后面放置特殊文档块即可；

  • 在将特殊文档块中的文本写入html和/或LaTeX输出文件之前，会对其解析，解析步骤：

  • A：Markdown格式被相应的html或特殊命令所取代

  • B：文档中的特殊命令将被执行，
    https://www.doxygen.nl/manual/commands.html

  • C：若一行以一些空格(whitespace)开头，后跟一或多个星号(*，asterisk)，
    然后可选地还有更多空格，则所有空格和星号都将被删除；

  • D：所有产生的空行都被视为段落分隔符(paragraph separator)；

  • E：为与文档类相对应的单词创建链接
    (除非单词前面有%；否则单词将不会被链接，且%符号将被删除)；

  • F：当在文本中找到某些模式时，将创建指向成员的链接；
    https://www.doxygen.nl/manual/autolink.html

  • G：文档中的Html tag将被解释并转换为LaTeX输出的LaTeX等效标记，
    https://www.doxygen.nl/manual/htmlcmds.html
